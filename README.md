# Culina — Culinary Recipe Management System 🍳

Цей репозиторій містить курсову роботу: повноцінну вебплатформу для управління кулінарними рецептами. Проєкт розроблено у вигляді монорепозиторію з розділенням на клієнтську та серверну частини, а також ядро утиліт.

## 🏗 Структура монорепозиторію

- **/frontend** — Клієнтська частина (Vanilla JS, HTML, CSS). Відповідає за відображення UI, візуалізацію даних та взаємодію з користувачем.
- **/backend** — Серверна частина. Побудована на базі сучасного фреймворку Fastify з використанням Prisma ORM для роботи з базою даних (PostgreSQL).
- **/core-utils** — Ядро проєкту. Ізольований модуль, який містить архітектурні рішення, незалежні від фреймворків.

## ✨ Основний функціонал (Features)

- **Управління рецептами (CRUD):** Створення нових рецептів, перегляд, повноцінне редагування зі збереженням змін.
- **Видалення даних:** Видалення окремих рецептів, а також масове видалення (bulk delete) декількох елементів одночасно з можливістю скасувати дію (undo).
- **Пошук та навігація:** Зручний текстовий пошук по базі рецептів та функція видачі випадкового рецепта (Random Recipe) за натисканням кнопки.
- **Статистика:** Динамічний підрахунок та відображення загальної кількості рецептів.

## 🚀 Архітектурні рішення та концепти

У проєкті імплементовано низку складних патернів проєктування для забезпечення масштабованості та чистоти коду:

- **Dependency Injection (DI) та Proxy:** Створено багаторівневу архітектуру HTTP-запитів, що повністю абстрагує UI від логіки авторизації та мережевих запитів. Реалізовано перехоплення помилок доступу (Unauthorized) з логікою оновлення токенів.
- **Logging Decorator:** Впроваджено систему логування за допомогою патерну "Декоратор" із підтримкою профілювання часу виконання, форматуванням та фільтрацією за рівнями.
- **Data Streams & Async Iterators:** Реалізовано пам'ятеефективну (memory-efficient) обробку великих масивів даних. Використано асинхронні генератори для порційного зчитування інформації без перевантаження оперативної пам'яті.

## 🛠 Як запустити проєкт локально

1. Клонування та встановлення залежностей:
-   git clone https://github.com/ВашЮзернейм/Culina.git
-   cd Culina
-   npm install

2. Налаштування оточення:
- Перейдіть у папку /backend.
- Скопіюйте файл .env.example та перейменуйте його на .env.
- Вкажіть ваші актуальні дані для підключення до бази даних у змінній DATABASE_URL.

3. Запуск серверної частини:
   npm run start:backend

4. Запуск клієнтської частини:
   npm run start:frontend
